What is Power Platform?

Power Platform is Microsoft's unified low-code/no-code suite that serves as both the experience layer (building apps, dashboards, portals) and the integration layer (connecting systems, automating workflows).

Key Insight: Dataverse is NOT separate from Power Platform—it IS Power Platform's data foundation. And Dynamics 365 is built ON Dataverse.

The Five Components

Component Purpose Typical Users
Power AppsBuild custom applications without codingBusiness analysts, citizen devs
Power AutomateWorkflow automation between systemsBusiness users, IT
Power BIBusiness intelligence and dashboardsAnalysts, managers
Power PagesExternal-facing portals and websitesWeb developers, IT
Copilot StudioBuild custom AI agents and copilotsAI developers, business users

Plus: Dataverse (The Data Foundation)

Dataverse provides:

  • Standard tables (Account, Contact, etc.)
  • D365 tables (Opportunity, Case, etc.)
  • BC Virtual Tables (Customers, Invoices, etc.)
  • Custom tables for your business data
  • Built-in security, relationships, and business rules

Connecting Power Platform to Your Stack

Business Central Integration

Option 1: BC Connector - Direct API access, real-time data

Option 2: Virtual Tables - BC data appears as Dataverse tables

// Power Automate: Create BC Sales Order
{
  "action": "BusinessCentral.CreateItem",
  "inputs": {
    "environment": "production",
    "company": "CRONUS",
    "entity": "salesOrders",
    "body": {
      "customerNumber": "{{bcCustomerNo}}",
      "externalDocumentNumber": "{{d365QuoteNo}}",
      "requestedDeliveryDate": "{{deliveryDate}}"
    }
  }
}

Microsoft Graph Integration

Via connectors: Outlook, Teams, SharePoint, OneDrive, Users

Via HTTP connector for full Graph API access:

// Power Automate HTTP action to Graph
{
  "method": "GET",
  "uri": "https://graph.microsoft.com/v1.0/me/messages",
  "headers": {
    "Authorization": "Bearer @{outputs('Get_Token')}"
  },
  "queries": {
    "$filter": "from/emailAddress/address eq '@{variables('CustomerEmail')}'",
    "$top": "10"
  }
}

SharePoint Integration

  • SharePoint lists as primary data source in Power Apps
  • Document library automation with Power Automate
  • Intelligent local caching for offline capability

Fabric Integration

  • Dataverse syncs to Fabric Lakehouse
  • Near real-time with Fast Fabric (<15 min latency)
  • Enables analytics across operational data

Power Automate vs Azure Logic Apps

Criteria Power Automate Azure Logic Apps
AudienceBusiness users, citizen devsDevelopers, IT pros
InterfaceVisual designer onlyVisual + code (VS/VS Code)
Best forM365/D365 workflowsEnterprise B2B, Azure-heavy
Connectors1,000+ (business focus)Azure ecosystem + enterprise
Custom codeLimited (expressions)Azure Functions integration
PricingPer-user or per-flowPer-execution (consumption)
RPADesktop flows includedNot included
Decision Rule: Start with Power Automate. Move to Logic Apps if you need Azure-scale, complex custom logic, or millions of executions.

Common Patterns

Pattern 1: 360° Customer View

Problem: Sales in D365 can't see BC financial data.

Solution: Enable BC Virtual Tables, add BC fields to D365 Account form.

Pattern 2: Credit Alert System

Problem: Sales creates deals for customers on credit hold.

Flow: Credit Status Sync (every 15 min)

1. Get BC customers where blocked or balance > creditLimit
2. For each: Find D365 Account, update bc_creditstatus
3. If status changed to blocked:
   - Teams: Send adaptive card alert
   - Dataverse: Create review Task

Pattern 3: Quote-to-Cash Automation

Problem: Won quotes manually re-entered in BC.

Flow: Quote Won → BC Order

Trigger: When Quote status = "Won"

1. Get Quote details and lines
2. Find/Create BC Customer
3. Create BC Sales Order
4. Create Order Lines (map products to items)
5. Update D365 Quote with BC order number
6. Notify via Teams and Email

Pattern 4: Document Auto-Classification

Problem: Users don't tag documents properly.

Flow: AI Document Classification

Trigger: File created in SharePoint

1. AI Builder: Extract text, classify type
2. Parse: Customer name, date, value
3. Find related Dataverse records
4. Move to correct library, update metadata
5. Create D365 Timeline activity
