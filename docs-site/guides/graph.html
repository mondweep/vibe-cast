What is Microsoft Graph?

Microsoft Graph is a unified REST API that provides a single endpoint to access data across the entire Microsoft 365 ecosystem.

Single Endpoint: https://graph.microsoft.com

Instead of learning separate APIs for Outlook, SharePoint, Teams, OneDrive, and Azure AD, you learn one API that gives you access to all of them.

What You Can Access

Service What You Can Do
MailRead, send, search emails
CalendarRead, create events, check availability
FilesAccess OneDrive and SharePoint documents
TeamsSend messages, manage channels
UsersRead profiles, manage groups
SearchSearch across all M365 content

Getting Started with Graph Explorer

The easiest way to explore Graph API is the official Graph Explorer - no code required.

  1. Go to Graph Explorer
  2. Sign in with your Microsoft account
  3. Try your first query: https://graph.microsoft.com/v1.0/me
  4. Click "Run query" to see your profile data
Important: Don't type "GET" in the query box - it's already there. Just paste the URL.

Common Queries

Your Profile

https://graph.microsoft.com/v1.0/me

Your Emails (Top 5)

https://graph.microsoft.com/v1.0/me/messages?$top=5&$select=subject,from,receivedDateTime

Your Calendar Events

https://graph.microsoft.com/v1.0/me/events

Your OneDrive Files

https://graph.microsoft.com/v1.0/me/drive/root/children

Search Emails

https://graph.microsoft.com/v1.0/me/messages?$search="project"

Your Teams

https://graph.microsoft.com/v1.0/me/joinedTeams

Query Parameters (OData)

Use these to filter and shape your API responses:

Parameter Description Example
$select Choose specific fields ?$select=displayName,mail
$top Limit results ?$top=5
$filter Filter results ?$filter=startswith(displayName,'John')
$search Search content ?$search="budget"
$orderby Sort results ?$orderby=receivedDateTime desc

Code Examples

Python

import requests

def get_my_emails(token):
    response = requests.get(
        "https://graph.microsoft.com/v1.0/me/messages",
        headers={"Authorization": f"Bearer {token}"},
        params={"$top": 10, "$select": "subject,from,receivedDateTime"}
    )
    return response.json()

emails = get_my_emails(access_token)
for email in emails["value"]:
    print(f"{email['subject']} - {email['from']['emailAddress']['address']}")

JavaScript (Node.js)

const axios = require('axios');

async function getMyEmails(token) {
    const response = await axios.get(
        'https://graph.microsoft.com/v1.0/me/messages',
        {
            headers: { Authorization: `Bearer ${token}` },
            params: { $top: 10, $select: 'subject,from,receivedDateTime' }
        }
    );
    return response.data;
}

const emails = await getMyEmails(accessToken);
emails.value.forEach(email => {
    console.log(`${email.subject} - ${email.from.emailAddress.address}`);
});

Common Permissions

When your queries fail with 403, you need to grant permissions:

Permission What It Allows
User.ReadRead your profile
Mail.ReadRead your emails
Mail.SendSend emails
Calendars.ReadRead your calendar
Files.ReadRead your OneDrive files

In Graph Explorer, click "Modify permissions" tab to grant these.
