Microsoft Copilot MCP Support Map

Microsoft has multiple Copilot products with varying MCP capabilities:

Product MCP Support Best For
GitHub Copilot (VS Code) Full - Local + Remote MCP Coding, file editing, terminal
Copilot Studio GA - Remote MCP only Business agents, workflows
M365 Copilot (Declarative) Preview - Remote MCP M365 extensions
copilot.microsoft.com None General Q&A only
Key Insight: If you only have access to copilot.microsoft.com (web chat), you have NO MCP or agentic capabilities. You need GitHub Copilot, Copilot Studio, or M365 Copilot licenses.

Option 1: GitHub Copilot + MCP (Best for Coding)

This is the closest to Claude Code + Claude Flow experience.

Setup MCP in VS Code

Create .vscode/mcp.json:

{
  "servers": {
    "github": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-github"],
      "env": {
        "GITHUB_PERSONAL_ACCESS_TOKEN": "${env:GITHUB_TOKEN}"
      }
    },
    "filesystem": {
      "command": "npx",
      "args": ["-y", "@anthropic-ai/filesystem-mcp-server", "/path/to/project"]
    }
  }
}

Use Agent Mode

  1. Open VS Code 1.99+
  2. Open Copilot Chat (Ctrl+Shift+I)
  3. Select "Agent" from mode dropdown
  4. Give high-level task: "Refactor auth module and run tests"
Agent Mode will: Create files, install dependencies, write code, run tests, and fix errors automatically.

Option 2: Copilot Studio + MCP

Best for building custom business agents that connect to external systems.

What You Can Do

  • Connect to remote MCP servers (cloud-hosted)
  • Build agents with custom tools
  • Access Microsoft Graph, Dataverse, SharePoint
  • Deploy to Teams, M365, web

Limitations for Coding

  • No local file system access (unlike Claude Code)
  • No terminal execution (unlike Agent Mode)
  • Primarily for business workflows, not coding

Setup

  1. Go to Copilot Studio
  2. Create new agent → Tools → Add Tool → MCP
  3. Provide your MCP server URL (must be cloud-hosted)

Option 3: M365 Declarative Agents + MCP

For extending Microsoft 365 Copilot with custom capabilities.

What You Can Do

  • Build agents inside M365 Copilot Chat
  • Connect to MCP servers for custom tools
  • Access SharePoint, Graph, M365 data
  • Display Adaptive Cards for rich UI

Setup with M365 Agents Toolkit

  1. Install M365 Agents Toolkit in VS Code
  2. Create new app → Declarative Agent
  3. Add MCP plugin to declarativeAgent.json
  4. Configure ai-plugin.json with MCP server URL

Recommendations by Use Case

Your Situation Best Option
Need Claude Code-like experience GitHub Copilot Agent Mode
Building business agents Copilot Studio + MCP
Extending M365 Declarative Agents + MCP
Need multi-agent swarms Build custom with Azure + MCP
Only copilot.microsoft.com Limited - no agentic capabilities
For true Claude Flow-like experience: Combine GitHub Copilot Agent Mode + Custom MCP servers + Azure Durable Functions (for orchestration) + GitHub Actions (for sandbox execution).
