Microsoft Skills

AI Knowledge

What it is: A repository of 130+ domain-specific knowledge files that enhance AI coding agents when working with Azure SDKs and Microsoft services.

Key insight: Skills teach AI agents about SDK patterns, not APIs themselves. Great for Azure, but gaps exist for Graph, D365, BC.

Useful skills:

  • azure-identity - Authentication patterns
  • mcp-builder - Build custom MCP servers
  • azure-search-documents - AI Search
  • m365-agents - Teams/Copilot bots

Dataverse

Platform

What it is: The unified data platform that Dynamics 365 is built on. Think of it as a managed database with built-in security, relationships, and business logic.

Key insight: When you use Dynamics 365, you're already using Dataverse. It's not a separate thing to set up.

Use it to:

  • Store operational business data
  • Create Virtual Tables to access BC data
  • Build Power Apps and automations
  • Apply row-level security
Read Full Guide →

Microsoft Graph

API

What it is: A single API endpoint to access everything in Microsoft 365 - emails, calendar, files, SharePoint, Teams, and more.

Key insight: One API to rule them all. Instead of learning 10 different APIs, learn Graph.

Use it to:

  • Read/send emails programmatically
  • Access SharePoint documents
  • Search across all M365 content
  • Manage users and groups
Read Full Guide →

Business Central

ERP System

What it is: Microsoft's cloud ERP for finance, supply chain, and operations. It has its own database and API - separate from Dataverse.

Key insight: BC data doesn't automatically appear in D365 CRM. You need Virtual Tables or sync to connect them.

Contains:

  • Customers, vendors, items
  • Sales orders and invoices
  • Payments and financial data
  • Inventory and purchasing
Read Full Guide →

Microsoft Fabric

Analytics Platform

What it is: A unified analytics platform for big data, data warehousing, and machine learning. For analyzing your business, not running it.

Key insight: Dataverse = operational (run the business). Fabric = analytical (analyze the business).

Use it to:

  • Analyze years of historical data
  • Build ML prediction models
  • Create unified data views across systems
  • Enable Copilot across all data
Read Full Guide →

SharePoint + Syntex

Content Platform

What it is: Document management and collaboration platform. Syntex adds AI to auto-classify and extract metadata from documents.

Key insight: SharePoint is accessed via Graph API. Syntex + Search + Purview make it actually usable at scale.

Use it to:

  • Store and collaborate on documents
  • Auto-classify content with AI
  • Apply governance and compliance
  • Link documents to D365 records

Solution Patterns

Proven architectural approaches to solve common integration challenges

Phase 1: Quick Wins

These solutions use existing licenses and can be implemented in days, not months.

360° Customer View

1-2 days
Problem: Sales can't see BC financial data (balance, credit limit, invoices) in D365 CRM.
Solution: Enable Virtual Tables for BC data in Dataverse
D365 Account Form
←
Virtual Table
←
BC Customer
Steps:
  1. Enable BC Virtual Table Provider in Power Platform Admin
  2. Connect to your BC environment
  3. Add BC fields (balance, credit limit) to D365 Account form
  4. Sales now sees BC data without leaving D365
Cost: Included in D365 license Skills: Power Platform admin

Credit Alert System

2-3 days
Problem: Sales creates opportunities for customers who are blocked or over credit limit in BC.
Solution: Sync credit status + Business Rule + Teams alert
BC Credit Status
→
Power Automate
→
D365 + Teams
Steps:
  1. Add bc_creditstatus field to Account in Dataverse
  2. Create Power Automate flow to sync BC blocked status
  3. Add Business Rule: Show warning when blocked
  4. Send Teams notification when status changes
Cost: Included in licenses Skills: Power Automate, D365 customization

SharePoint Information Architecture

1-2 weeks
Problem: Documents are scattered, inconsistently tagged, and impossible to find.
Solution: Hub sites + Content Types + Search configuration
Hub Sites
+
Content Types
+
Search Bookmarks
Steps:
  1. Define taxonomy (departments, doc types, status)
  2. Create hub site structure per department
  3. Create standardized content types with required metadata
  4. Configure Search bookmarks for common queries
Cost: Included in M365 license Skills: SharePoint admin

Phase 2: Enhanced Integration

More sophisticated solutions requiring some development effort.

Quote-to-Cash Automation

1-2 weeks
Problem: Won quotes in D365 are manually re-entered as sales orders in BC.
Solution: Power Automate flow triggered on quote win
D365 Quote Won
→
Power Automate
→
BC Sales Order
Flow logic:
  1. Trigger: Quote status changes to "Won"
  2. Find/create customer in BC
  3. Create sales order with line items
  4. Update D365 with BC order number
  5. Notify finance team via Teams
Cost: Power Automate Premium Skills: Power Automate, BC API

AI Document Classification

2-3 weeks
Problem: Users don't fill in metadata. Documents have no context.
Solution: Train Syntex models to auto-extract metadata
Document Upload
→
Syntex AI
→
Auto-tagged
Steps:
  1. Identify high-volume document types (contracts, invoices)
  2. Train document understanding model with examples
  3. Define extraction rules (vendor, amount, date)
  4. Apply model to libraries
  5. Documents now auto-tagged on upload
Cost: Syntex license Skills: Syntex model training

Phase 3: Fabric + Copilot

Enterprise-scale analytics and AI-powered experiences.

Campaign ROI with Fabric

3-4 weeks
Problem: Marketing can't trace campaigns to actual paid revenue in BC.
Solution: Fabric Lakehouse unifies D365 + BC data
D365 Campaigns
→
Fabric OneLake
←
BC Payments
Architecture:
  1. Sync D365 campaigns, leads, opportunities to Fabric
  2. Sync BC invoices and payments to Fabric
  3. Create unified view joining campaign → lead → opportunity → invoice → payment
  4. Build Power BI dashboard showing true ROI
  5. Enable Copilot: "Which campaign had the best ROI?"
Cost: Fabric capacity (~$300+/mo) Skills: Data engineering, SQL, PySpark

Copilot Across All Data

4-6 weeks
Problem: Users need to query data across D365, BC, and SharePoint using natural language.
Solution: Fabric semantic model + Copilot
Natural Language
→
Copilot
→
Fabric Semantic Model
Capabilities:
  • "Show me customers with overdue invoices and open opportunities"
  • "What's our quote-to-cash cycle time by product?"
  • "Predict which deals will close this month"
  • "Find contracts expiring in the next 30 days"
Cost: Fabric + Copilot licenses Skills: Semantic modeling, data architecture
