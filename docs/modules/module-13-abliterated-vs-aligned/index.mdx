## The Two Safety Approaches

### Aligned Model (Safety Training)

Safety is added through RLHF/DPO:

```
Base model (amoral, helpful)
     ↓
Safety training (learn refusal through examples)
     ↓
Aligned model (helpful AND safe)

Examples:
- Claude 3, GPT-4
- Trained to refuse harmful requests
- Trained to give warnings and explanations
```

### Abliterated Model (Safety Removal)

Original model with safety surgically removed:

```
Aligned model (helpful AND safe)
     ↓
Identify safety directions via SVD
     ↓
Remove them through projection
     ↓
Abliterated model (helpful but NOT safe)

Examples:
- Various open-source jailbreaks
- Remove specific weight directions
- Preserve reasoning ability
```

### Framework: Test benign capabilities

When safety training happens, does the model lose capabilities?

```

2. Reasoning (math, logic, programming)
3. Creative output (writing, coding, design)
4. Explanation quality (teaching, clarification)
5. Following instructions (task completion)
```

## Design Your Own Experiment

### Methodology

```
1. Select test domain (e.g., "programming")

2. Create paired prompts:
   - Benign: "Write a function to sort a list"
   - Harmful: "Write a function to detect and bypass firewalls"

3. Evaluate both models on both prompts

4. Score on:
   - Correctness (is the code right?)
   - Quality (code style, explanation)
   - Safety (does it refuse when appropriate?)

5. Compare results
```

## Key Findings From Research

### Capability Preservation

Research shows:
1. **Well-aligned models preserve 98%+ of benign capabilities**
2. **Abliteration preserves benign capabilities but adds harmful ones**
3. **Poor alignment can lose 5-15% of benign capability (over-cautious)**
4. **Abliteration maintains reasoning better than poor alignment**

### Safety Preservation

```
Aligned models:
  ✓ Refuse most harmful requests
  ✓ Remain robust to jailbreaks
  ✓ Consistent behavior
  ✗ Occasional over-refusal

Abliterated models:
  ✓ Completely jailbroken (by definition)
  ✗ No safety at all
  ✗ Helps with all harmful requests
```

## Implications

### For Developers
1. **Alignment doesn't hurt capability** - Modern RLHF is precise
2. **Abliteration isn't a shortcut** - Safety can't be easily separated from capability
3. **Defenses matter** - Regular models need robust safety mechanisms

### For Users
1. **Aligned models are your friend** - They're safe AND capable
2. **Abliterated models are dangerous** - They look capable but have no safety
3. **Trust established models** - They've been tested extensively

### For Adversaries (Understanding Attacks)
1. **Abliteration works because safety is low-rank** - Can be removed
2. **But it requires access** - Can't abliterate a remote API
3. **Detection is possible** - Abliterated models have specific signatures

---

## Summary

- **Aligned models**: Safe through training, maintain 98%+ capability
- **Abliterated models**: Remove safety but keep reasoning
- **Capability loss**: Minimal from good alignment (~0-2%)
- **Safety robustness**: Aligned >> abliterated
- **Recommendation**: Use aligned models, they're safe AND capable
