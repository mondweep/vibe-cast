# Paperclip Customer Support Demo

**Project**: Multi-Agent Customer Support Triage System  
**Purpose**: Demonstrate autonomous agent coordination handling real-world support scenarios  
**Tech Approach**: Paperclip agent orchestration with Next.js/React frontend hosted on Netlify  
**Methodology**: BHIL (Barry Hurd Innovation Lab) AI-First Development

## Documentation Structure

- `docs/prd/` — Product Requirements Documents
- `docs/spec/` — Technical Specifications
- `docs/adr/` — Architecture Decision Records
- `docs/tasks/` — Implementation Task Breakdowns
- `docs/prompts/` — Prompt Registry & Versioning

## Key Conventions

- All artifacts have unique IDs (PRD-NNN, SPEC-NNN, etc.)
- Specifications are the source of truth
- Quantified acceptance criteria only
- Agents answer spec gaps; don't fix specs in chat
