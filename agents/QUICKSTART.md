# Finabeo Marketing Agent - Quick Start (5 minutes)

## ✅ What's Been Built

A **production-ready, multi-agent marketing system** using Microsoft Agent Framework:

```
┌─────────────────────────────────────────────────────────┐
│ Market Research Agent                                   │
│ → Researches trends, identifies pain points             │
└──────────────────┬──────────────────────────────────────┘
                   │
┌──────────────────▼──────────────────────────────────────┐
│ Finabeo Alignment Agent                                 │
│ → Maps services to market needs (0.0-1.0 scoring)       │
└──────────────────┬──────────────────────────────────────┘
                   │
┌──────────────────▼──────────────────────────────────────┐
│ Content Generation Agent                                │
│ → Creates LinkedIn, Twitter, Instagram, Blog content    │
└─────────────────────────────────────────────────────────┘
                   │
                   ▼
           📁 Output (JSON)
```

## 🚀 Run in 5 Minutes

### Step 1: Set Your API Key

```bash
# On Mac/Linux:
export FOUNDRY_API_KEY="your-foundry-api-key-here"

# On Windows PowerShell:
$env:FOUNDRY_API_KEY="your-foundry-api-key-here"
```

(Use your actual API key from Azure Portal → Foundry Project → Keys and Endpoint)

### Step 2: Navigate to Project

```bash
cd agents/FinabeoMarketingAgent
```

### Step 3: Build & Run

```bash
# Build
dotnet build

# Run
dotnet run
```

## 🐛 Troubleshooting

**Issue**: "Invalid Foundry configuration"
- Check FOUNDRY_API_KEY is set: `echo $FOUNDRY_API_KEY`
- Verify endpoint in appsettings.json

**Issue**: Build fails with "nuget not found"
- Restore packages: `dotnet restore`
- Check .NET version: `dotnet --version` (need 8.0+)

**Issue**: Agent returns mock data
- Normal! Agents fall back to mock data when testing
- Real data flows when connected to Foundry
