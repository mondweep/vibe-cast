What is Agentic AI?

🔬

Reasoning

AI agents break down complex problems into manageable steps, thinking through solutions systematically before acting.

🛠

Tool Use

Agents extend their capabilities by using external tools — APIs, databases, code execution, web search, and more.

🚀

Autonomy

Unlike simple chatbots, agentic systems execute multi-step workflows independently, adapting as they go.

🤝

Collaboration

Multiple specialized agents can work together — one plans, another codes, another tests — creating powerful orchestration.

Key Agentic Techniques

The building blocks of modern AI agent systems

01

ReAct Pattern

Interleave reasoning (thinking) and acting (tool use) in a loop. The agent thinks about what to do, takes an action, observes the result, and repeats.

Thought: I need to find the weather data
Action: search("Nagaon weather today")
Observation: Current temp: 24°C, Humidity: 78%
Thought: Now I can answer the user
Action: respond("The weather in Nagaon...")
02

Task Decomposition

Break complex goals into a tree of sub-tasks. Each sub-task is small enough for a single agent call, making the overall problem tractable.

Goal: "Build a weather dashboard"
├── Plan architecture
├── Create API integration
│   ├── Research weather APIs
│   └── Write fetch logic
├── Build UI components
│   ├── Temperature widget
│   └── Forecast chart
└── Deploy to production
03

Tool Use & Function Calling

Give agents access to external tools — search engines, calculators, code interpreters, databases — and let the LLM decide when and how to use them.

tools = [
  { name: "web_search", desc: "Search the web" },
  { name: "run_code", desc: "Execute Python" },
  { name: "read_file", desc: "Read a file" },
  { name: "write_file", desc: "Create/edit files" }
]
// LLM chooses the right tool for each step
04

Memory & State

Agents maintain short-term context (conversation) and long-term memory (vector stores) to build on past interactions and knowledge.

Short-term: Current conversation context
Long-term:  Vector database of past sessions
Episodic:   "Last time user preferred Python"
Semantic:   "RAG best practices document"
Procedural: "Steps to deploy on Netlify"
05

Multi-Agent Systems

Multiple specialized agents collaborate — a planner delegates to a coder, whose output is reviewed by an evaluator, creating emergent intelligence.

Orchestrator → Planner Agent
  └→ "Break down the user's request"
Planner → Coder Agent
  └→ "Implement step 3: API endpoint"
Coder → Reviewer Agent
  └→ "Check for bugs and improvements"
Reviewer → Tester Agent
  └→ "Write and run test cases"
06

Safety & Guardrails

Responsible agentic AI includes permission boundaries, human-in-the-loop checkpoints, output validation, and defense against adversarial inputs.

Guardrails:
  ✓ Permission levels (read / write / execute)
  ✓ Human approval for risky actions
  ✓ Input sanitization against injection
  ✓ Output validation and fact-checking
  ✓ Rate limiting and cost controls
  ✓ Audit logging of all actions
